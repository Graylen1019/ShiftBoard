ShiftBoard has four views. The Shift Overview shows current shift status, time elapsed, food cost variance entered by the manager, and a task completion counter. The Task Checklist shows shift tasks grouped by category (opening, mid-shift, temperature checks, closing), each with a checkbox, timestamp, and notes field. Tasks can be marked complete, skipped with a reason, or flagged. The Waste Log is a running log of waste events: the manager selects a category, enters quantity and unit, and adds an optional note. Entries are timestamped automatically. The Shift Report is generated when the manager closes the shift: it compiles task completion rate, waste totals by category, flagged items, and food cost variance into a readable summary that can be copied or downloaded.
The feature that lifts this above a basic CRUD app is the 7-day trend view, built in Phase 2. After enough shifts, the dashboard shows a Recharts line chart of food cost variance and waste totals over the past week. This is something no paper-based system can provide and is genuinely useful.
